# dpyc-crypto

A small Rust WebAssembly component providing the native secp256k1 + AES
primitives that the WASI CPython operator component cannot supply itself
(no native extension modules in the bundled interpreter).

Composed alongside the Python component by `wasmcp compose`; the Python
`coincurve`/`cryptography` stub modules adapt the wheel's calls onto this
interface. Key derivation (HKDF for the vault) stays pure-Python — only the
ciphers and curve ops live here.

## Interface (`dpyc:crypto/ops`, see `wit/world.wit`)

- `ecdh-nip04` — NIP-04 ECDH shared secret (raw X-coordinate, 0x02 even-Y convention)
- `xonly-pubkey` — x-only pubkey from a secret key
- `schnorr-verify` — BIP-340 over a 32-byte message (Nostr event id); uses
  k256 `verify_raw` (NOT `verify`, which SHA-256-hashes the message first)
- `aes256-cbc-{encrypt,decrypt}` — PKCS7 (NIP-04 payload)
- `aes256-gcm-{encrypt,decrypt}` — nonce + aad (vault rows)

## Structure

- `src/crypto_core.rs` — pure, native-testable crypto. No WIT/wasm deps.
- `src/lib.rs` — WIT binding layer (wasm-gated) + native tests.
- `testdata/vectors.json` — ground truth generated by the tollbooth-dpyc wheel
  (coincurve + cryptography) plus the real Tollbooth Sample bootstrap event.

## Verification

`cargo test` (host target) — 7/7 pass, byte-for-byte against the wheel:
ECDH matches + symmetric, x-only pubkey matches, NIP-04 decrypt of wheel
ciphertext, AES-GCM decrypt of the real wheel vault-row format (AAD-bound;
wrong AAD rejected), CBC/GCM round-trips, and schnorr-verify of the **real
Authority signature** on the live bootstrap event (tampered id rejected).

Build the component: `cargo component build --release`
→ `target/wasm32-wasip1/release/dpyc_crypto.wasm` (~194 KB).

Invoke cargo/rustup by full path (`~/.cargo/bin/...`) — this host also has a
Homebrew rust on PATH that lacks wasm targets.
